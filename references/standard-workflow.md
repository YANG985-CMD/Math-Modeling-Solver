# Standard Workflow

## Competition-Time Workflow

### First 0-2 Hours

- Read the whole problem once
- Mark each sub-question
- Identify likely task type
- Decide whether one shared model can cover multiple sub-questions
- Split roles: modeling, coding, writing

### Middle Phase

- Clean and inspect data
- Build the first-pass model
- Get one runnable solution early
- Compare at least one alternative if risk is high
- Save tables and plots for writing

### Final Phase

- Write abstract last
- Cross-check variable names, units, and figure references
- Add sensitivity or robustness analysis
- State limitations honestly

## Model-Building Workflow

1. Problem restatement
2. Assumptions
3. Symbol definition
4. Data preprocessing
5. Model selection
6. Solving or training
7. Validation and sensitivity analysis
8. Result interpretation
9. Limitations and improvement directions
