# Competition Timeline

## 24-Hour Sprint

- 0-3h: triage, split work, choose route
- 3-12h: build and verify first-pass model
- 12-18h: improve model and generate figures
- 18-24h: write, cross-check, polish

## 12-Hour Compressed Mode

- 0-2h: triage and route lock
- 2-7h: one main model plus one backup check
- 7-12h: write and validate

## 6-Hour Rescue Mode

- 0-1h: choose the simplest defensible route
- 1-4h: get one runnable model with interpretable output
- 4-6h: write minimal but coherent paper
